## Summary

Read-only dashboard card and API route added for generated weekly Money OS review artifacts.

## Route Added

```
GET /api/ghoti/money/weekly-review/latest
```

Source: `01_projects/dashboard_mvp/server.js`

### Behavior

- Scans `05_logs/money_reviews/` for run dirs
- Selects latest run by lexicographic sort (run IDs are timestamped)
- Reads `weekly_review.json` — machine-readable summary
- Reads `decisions_recommended.jsonl` — tolerant JSONL parser (malformed lines → warning count)
- Returns top 3 decision candidates, candidate counts by category, artifact paths
- Zero-state: returns `{ ok: true, status: "zero_state", runs_total: 0 }` if dir missing or empty
- Safety check: warns if safety_flags has unexpected true values
- Never mutates files; never triggers live actions; never executes model output

### Response Fields

| Field | Description |
|-------|-------------|
| status | "found" or "zero_state" |
| runs_total | total run dirs found |
| latest_run_id | run ID of latest review |
| created_at | ISO timestamp of latest review |
| week_start / week_end | review period |
| source_counts | tracker_experiments + money_run_summaries |
| total_experiments | total experiments in tracker |
| total_money_runs | money run dirs found |
| top_candidates | top 5 from weekly_review.json |
| next_local_actions | from weekly_review.json |
| warnings | parse warnings + missing file warnings |
| safety_flags | from weekly_review.json |
| approval_required_for_public_actions | always true |
| candidate_counts | counts by category from decisions_recommended.jsonl |
| top_decision_candidates | top 3 parsed from JSONL |
| total_decision_candidates | total parsed JSONL lines |
| artifact_paths | relative path + exists flag for each artifact file |
| artifact_dir | relative path to latest run dir |
