## 1-Day Build

Goal: create a minimum sellable asset without new tooling.

- Pick one product: prompt library or automation checklist pack.
- Create product outline.
- Use Gemma to draft sections and compress existing Ghoti notes.
- Manually review every item.
- Produce a markdown bundle and simple README.
- Draft listing copy, but do not publish.

Suggested 1-day MVP:

- `AI Operator Starter Kit`
- Contents:
  - setup checklist
  - safety gate checklist
  - worker card template
  - local Gemma task list
  - prompt handoff rules
  - money workflow ideation worksheet

## 3-Day Build

Goal: turn the MVP into a more credible product.

- Add examples and before/after workflows.
- Add a quick-start guide.
- Create a simple product page draft.
- Create 10 short-form content ideas to promote it.
- Draft Whop/Gumroad-style listing copy for human review.
- Add clear disclaimers and "no income guarantee" wording.

## 7-Day Build

Goal: create a small product library or first productized service offer.

- Bundle 2 to 3 related products.
- Add sample filled templates.
- Create a lightweight landing page plan.
- Add FAQ, refund policy draft, and support boundaries.
- Create a 14-day content calendar.
- Manually test whether a stranger can understand the product in 5 minutes.

## Required Assets

- Product name.
- Buyer persona.
- Promise statement.
- Scope and non-scope.
- Template files.
- Example filled output.
- README.
- Listing copy.
- Screenshots/mockups if needed.
- Risk/disclaimer note.
- Manual publish checklist.

## How Gemma Can Draft Cheaply

Gemma can:

- summarize Ghoti docs into buyer-friendly lessons
- draft checklist items
- create product outlines
- rewrite internal notes into clean language
- generate title and subtitle options
- create first-pass FAQs
- draft content hooks and post outlines

Gemma should not:

- make final income claims
- publish listings
- decide pricing alone
- connect to Whop or payment platforms
- generate legal terms as final advice
- automate customer communication

## How Claude Code Can Build Later

Claude Code can:

- create `14_context/money_workflows/product_templates/`
- build markdown product templates
- add local Gemma artifact formats
- create a static landing page draft if requested
- add validation scripts
- update dashboard read-only views later

Claude Code should not:

- create live Whop products
- connect payment accounts
- post content
- automate outreach
- use paid media tools
- wire Paperclip/OpenClaw/n8n into sales workflows yet

## How Codex Can Audit Positioning

Codex should review:

- buyer clarity
- claim accuracy
- product differentiation
- legal/TOS risk
- pricing assumptions
- whether the product can be built with current repo-local tools
- whether Gemma output needs human/Claude review

## What Should Not Be Automated Yet

- Whop account actions.
- Product upload.
- Payment setup.
- Checkout configuration.
- Refund handling.
- Customer messaging.
- Affiliate link generation.
- Social posting.
- Review/testimonial generation.

## First MVP Recommendation

Build `AI Operator Starter Kit` first because it uses existing Ghoti artifacts, requires no external tools, and proves the money-workflow loop:

1. extract value from local project memory
2. use Gemma for cheap drafting
3. use Codex for audit
4. use Claude Code for templates
5. user approves product positioning before any store action
