## Purpose

Design the next safe Gemma-powered runner for turning local money-video transcripts or notes into business-model summaries, product candidates, content angles, distribution plans, and risk reviews. This is an implementation spec only. Codex did not edit runtime code or run Gemma/Ollama.

## Proposed Future File

Suggested implementation target:

`03_scripts/money_video_to_money_runner.py`

Alternative if Claude prefers keeping scripts grouped:

`03_scripts/money_workflow_video_to_money.py`

## Input Rules

Allowed input:

- repo-local `.md` or `.txt` transcript
- repo-local notes file
- repo-local pasted outline from a video/course/user idea
- compact output from `local_brain_router.py --task compress_context`

Forbidden input/action:

- URL fetch
- YouTube downloading
- scraping
- browser automation
- external API
- paid tool calls
- live account access
- comments scraping
- platform data collection

The runner must not claim to "watch" a video. It can only process text provided locally by the user or generated by an approved local artifact workflow.

## Output Rules

The runner should write artifacts only under:

`05_logs/money_workflow_runs/<run_id>/`

Required output files:

- `request.json`
- `source_excerpt.md` or `source_excerpt.txt`
- `gemma_prompt.txt`
- `gemma_response.txt`
- `business_model_summary.md`
- `product_candidates.md`
- `content_angles.md`
- `distribution_plan.md`
- `risk_review.md`
- `run_summary.json`

Optional output files:

- `experiment_candidates.jsonl`
- `approval_checklist.md`

The first N+3.18 implementation should not auto-append to `14_context/money_workflows/experiment_tracker.jsonl`. The operator should review candidates first.

## Runner Flow

1. Parse CLI args.
2. Validate input path is inside repo root.
3. Validate extension is `.md` or `.txt`.
4. Read and clip source text to `--max-chars`, default 12000.
5. Write `request.json` and `source_excerpt`.
6. Build a deterministic Gemma prompt from the source excerpt.
7. Run local Ollama/Gemma only if available.
8. Save raw Gemma output as `gemma_response.txt`.
9. Extract or copy structured sections into markdown artifacts.
10. Write `run_summary.json`.
11. Print run ID and output folder.

## CLI Shape

```powershell
python 03_scripts/money_video_to_money_runner.py --input 14_context/money_workflows/video_intake/example_notes.md --max-chars 12000 --dry-run
python 03_scripts/money_video_to_money_runner.py --input 14_context/money_workflows/video_intake/example_notes.md --max-chars 12000
```

Recommended args:

- `--input <repo-local path>` required
- `--max-chars <number>` optional, default 12000
- `--dry-run` optional, write request/source/prompt only and skip model call
- `--output-root 05_logs/money_workflow_runs` optional, but must stay repo-local

## Prompt Requirements

Gemma prompt should ask for:

- main claim
- monetization mechanism
- required skills
- startup capital required
- time required
- tools required
- proof quality
- legal/TOS concerns
- risks
- minimum viable experiments
- productizable templates
- content angles
- email lead magnet ideas
- distribution channels
- approval gates

Gemma prompt must also say:

- do not invent facts
- label uncertainty
- no financial/legal/tax advice
- no guarantee of revenue
- no scraping/posting/outreach
- output is for internal planning only

## No Auto-Append Yet

The first runner should not mutate the experiment tracker. It can write `experiment_candidates.jsonl` inside the run folder, but human review is required before appending to the canonical tracker.

Later approved flow:

1. Runner creates candidates.
2. Operator reviews candidates.
3. `money_workflow_new_experiment.py` appends selected experiment.

## PASS Criteria

PASS means:

- local input only
- no URL fetch
- no external API
- artifacts written only under `05_logs/money_workflow_runs/<run_id>/`
- no model output executed
- no canonical tracker mutation
- no live account action
- no posting/selling/outreach/payment

## Blockers

Block implementation if:

- it tries to fetch URLs
- it shells out to browsers, Docker, CUA, or scraping tools
- it appends to the canonical tracker without review
- it writes outside repo root
- it treats Gemma output as truth without uncertainty labels
- it generates fake proof, spam scripts, or deceptive claims

## Recommendation

Build this as a dry-run-first artifact generator. Let Gemma produce cheap first drafts, then let the operator and Codex/Claude decide what becomes a real experiment.
