# Repo Classifications

Short version of the current repo stance.

## Claw Code

- role: reference / later experiment
- not the current foundation
- current local path is temporary and outside the permanent root
- current center of gravity looks Rust-first, though the repo is mixed and its docs are internally inconsistent

## OpenHarness

- role: later harness candidate
- not first install
- worth revisiting once Python and local model runtime are in good shape

## oh-my-codex

- role: optional workflow helper
- useful only if Codex stays central

## OpenScreen

- role: side utility
- useful for demos, not assistant architecture

## awesome-design-md

- role: optional design/prompt reference
- useful for UI-heavy project work, not core infra

## sherlock

- role: side tool only

## RuView

- role: unrelated to the core assistant stack
