# Desktop Bridge Foundation

## Purpose

The user wants real local control eventually: switch apps, click, type, run commands, copy and paste, and wait for approvals or replies. This batch does not pretend that full control already exists. It creates the first honest bridge layer toward it.

## What Exists Now

- local browser demo only
- no desktop controller yet
- no app switching, cursor movement, or native app automation yet

## What This Foundation Should Provide First

- local desktop-control status that the dashboard can show clearly
- safe environment checks for the Windows shell and local launcher path
- visible operator-console reporting instead of vague claims
- explicit confirmation that real desktop control is still missing

## Likely Later Path

- Windows-Use and Windows-MCP exploration
- approval loop before risky actions
- observation loop for state checking
- audit log for what happened
- kill switch for local control
- eventually agent-to-tool orchestration once the boundaries are clear
