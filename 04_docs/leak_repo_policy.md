# Leak Repo Policy

## Purpose

Keep the foundation anchored to official, reviewable, and defensible reference material.

## Policy

- official Claude Code is cleaner reference material than leak or extraction collections
- leak-adjacent repos should not be treated as core dependencies
- provenance matters more than novelty

## Why

- malware and supply-chain risk
- unclear authorship and licensing
- stale or misleading snapshots
- bad foundation for a publishable system

## Current Application

- `anthropics/claude-code` is acceptable reference material
- `chauncygu/collection-claude-code-source-code` is excluded from this batch and should not be treated as foundation material

## If Reviewed At All

- keep it quarantined
- keep it reference-only
- do not merge code or assumptions into core without separate verification
