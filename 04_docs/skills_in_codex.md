# Skills In Codex

## What A Skill Is Here

A skill is a reusable instruction bundle that tells Codex how to handle a narrow kind of task more reliably than ad hoc prompting.

## ChatGPT-Side Vs Codex-Side

- ChatGPT-side skills or custom behavior are not automatically available in Codex
- Codex can only use skills that exist in the current Codex environment
- repo-local skills would be useful when this project develops repeatable workflows that should not depend on memory alone

## When To Tell Codex To Use A Skill

- when the task clearly matches an available skill
- when browser automation needs a real browser toolchain
- when GitHub or Notion work has a known skill path
- when document or report generation needs a stable workflow

## When Repo-Local Skills Would Help

- approval-gated GitHub workflows
- browser smoke tests and review flows
- internship and personal-ops pack generation
- repo intake and extraction reviews

## Skills Likely To Matter Later

- Playwright CLI for browser flows
- GitHub workflow skills for PR or issue handling
- Notion planning skills for structured workspace updates
- report generation skills for repeatable exports and case studies
