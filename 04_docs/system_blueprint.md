# System Blueprint

## Primary Goal

This system is an execution-first AI control framework for legitimate business, documentation, research, and owned-account workflows. It is meant to turn ideas, guides, repos, and source material into plans, experiments, assets, reports, and tracked execution while staying approval-gated and inspectable.

## Core Layers

- control layer: Continue for local context and rules, Codex for execution and Git workflow
- memory/handoff layer: `14_context` plus scoped project and agent memory
- runtime/task layer: file-backed task lifecycle and local runtime commands
- approval layer: explicit approval policy, task approval state, and human checkpoints
- AI council/model-routing layer: provider profiles, council planning, and routing policy examples
- integrations layer: future GitHub, Notion, browser, app, and repo integrations
- documentation/report layer: templates, report scaffolds, exports, runbooks, and summaries

## Current Reality

Already in place: Git-backed workspace, durable handoff files, approval skeleton, memory architecture, Codex workflow docs, runtime MVP, and initial council/workflow/report scaffolding.

Still missing: live provider integrations, browser or app control, deeper lifecycle semantics, richer approval flow, and real Notion integration.

## Near-Term Priorities

- council planning scaffolding
- runtime lifecycle deepening
- report and document automation
- approval flow refinement
- integration groundwork
