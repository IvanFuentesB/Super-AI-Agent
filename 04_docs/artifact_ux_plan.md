# Artifact UX Plan

## Purpose

Generated outputs should be immediately usable from the dashboard, not hidden as files that only make sense if the user goes digging through folders.

## What The Dashboard Should Do

- show a recent artifact list
- preview markdown and text outputs in a readable way
- open the selected artifact in the default local app
- reveal the selected artifact in Explorer

## Why Raw Editor View Is Not Enough

- generated packs are more useful when the user can scan them quickly before deciding what to edit
- many outputs are review documents, not just implementation artifacts
- a dashboard should shorten the path from generation to review, not send the user back into file hunting

## Future Artifact UX

- richer markdown rendering
- artifact filters by type or workflow
- pinning or favoriting important outputs
- approval-aware publish or export actions later
