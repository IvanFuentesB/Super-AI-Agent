# Career-Ops Fit

## Why It Is Relevant

`santifer/career-ops` is relevant because it treats job search as a structured operating system instead of a pile of ad hoc applications. That matches this repo's execution-first direction.

## What To Borrow Conceptually

- fit scoring before action
- one source of truth for applications and status
- tailored CV and case-study preparation
- human-in-the-loop review before submission
- explicit separation between evaluation, drafting, and action

## What Not To Copy Blindly

- Claude-specific runtime assumptions
- heavy Node, Go, and Playwright stack choices
- any direct application-filling or outreach flow
- repo-specific modes and commands that are tightly coupled to that project

## How It Helps Internships And Job Search

- gives a model for triaging roles instead of applying blindly
- helps frame internship packs, CV changes, and portfolio proof
- gives useful ideas for tracking, scoring, and case-study preparation

## What Should Stay Reference Only

- portal automation details
- large custom dashboard implementation
- batch worker architecture
- direct application or outreach flows

## What Can Become Part Of This Repo Later

- internship fit scoring
- compact application tracking
- portfolio and case-study packaging
- evidence and proof-point memory for career ops
