# N+6.44A Local Agent Command Center Simulation

## Outcome

Ghoti now has one local command-center CLI that connects:

- source-linked, verified shared memory and Obsidian views
- supervised multi-agent scenario plans
- non-overlapping file ownership and dependency waves
- Agent Arena-shaped queue/status previews
- a Paperclip-compatible company/team/work-item planning preview

This is the first coherent operator-facing composition layer. It is a real
planner and simulator, not a live agent launcher.

## Business Experiments

The included content and ecommerce scenarios help prepare evidence, drafts,
margin assumptions, risk reviews, and human decisions. The system does not
guarantee income and cannot publish, open stores, contact suppliers, buy
anything, spend on ads, or access accounts.

## Repo Pattern Use

- Paperclip informs the company, department, work-item, and approval preview.
- Claude-swarm and am-will/swarms inform coordinator/worker and dependency-wave
  planning.
- Agent Arena provides the simulation/status shape.
- Ghoti shared memory and Obsidian views provide source-linked context.

No third-party code is copied or executed. Paperclip remains a Docker-dependent
future integration. Hermes Paperclip remains a future adapter.

## Safety Boundary

Full computer-use remains blocked. No agent process, mouse click, keyboard
input, browser control, account action, posting, purchase, payment, ad spend,
supplier contact, auto-submit, MCP, hook, Docker service, or external repo is
launched.

Every scenario is simulation-only, includes a human approver, blocks risky
actions, and reports file-ownership conflicts before any future execution.
