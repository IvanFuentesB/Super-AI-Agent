# Diagram Update Rule

Public diagrams must stay aligned with audited current behavior.

## Rules

- Do not show Hermes fully installed unless verified.
- Do not show Hermes Codex provider support unless verified.
- Do not show UI-TARS click, type, hotkey, or desktop control as enabled.
- Do not show external repo runtime wiring unless a later audited milestone enables it.
- Do not show live account actions, posting, uploading, trading, or money movement.
- Do not show Telegram connected unless the user manually configures it later.
- Keep approval gates visible in every workflow that could become risky.
- If README diagrams change, update public docs and the latest report in the same branch.
