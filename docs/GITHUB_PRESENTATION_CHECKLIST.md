# GitHub Presentation Checklist

Use this checklist for Ghoti and the broader portfolio.

## Repository Page

- Confirm the repository name clearly includes or references Ghoti later.
- Add branded hero/banner imagery.
- Add a clear description and topics.
- Confirm the license sidebar does not imply open source for Ghoti.
- Link to architecture docs and current limitations.

## Templates To Add Later

- Issue template with expected behavior, actual behavior, safety impact, and
  private-data warning.
- Pull request template with tests, docs, security, and approval-gate checklist.

## Public Review Notes

- Public visibility is for demonstration and review only.
- Curated images are copies; raw user imports stay ignored.
- Human approval is required for repo/profile presentation decisions.
