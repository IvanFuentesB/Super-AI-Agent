# GitHub Profile And Repo Upgrade Playbook

This is a portfolio/repo presentation side lane. It supports Ghoti and the
user's broader engineering portfolio without replacing normal Codex plus Claude
Code implementation/audit workflow.

## Strategy

1. Pin the right 6 repos: NXP, SDA, Doodle, Ghoti, best school/engineering repo, one polished future project.
2. Add repo topics: robotics, mechatronics, embedded-c, computer-vision, opencv, mqtt, pytorch, mediapipe, s32k144, autonomous-vehicle.
3. Add demo GIFs for Doodle and SDA.
4. Add GitHub Actions checks: Python compile/test, markdown link checker, Node check for Ghoti.
5. Create releases: v1.0-portfolio for SDA, Doodle, Ghoti.
6. Add clean repo descriptions and website links.
7. Open real issues for future improvements: HSV calibration, MQTT reconnect, sample dataset docs, Doodle training improvements.
8. Add LICENSE files where appropriate. MIT for fully owned small repos if safe; proprietary/all-rights-reserved for Ghoti for now.
9. Add `docs/architecture.md` to each strong repo.
10. Contribute small PRs to external repos.
11. Answer issues/discussions.
12. Use Issues/Discussions for architecture decisions.
13. Add `.github/ISSUE_TEMPLATE` and `PULL_REQUEST_TEMPLATE.md`.
14. Make one repo genuinely useful.
15. Do consistent real commits, no fake spam.

## Top 5 Fastest Wins

1. Pin repos.
2. Add topics.
3. Add demo GIFs.
4. Add Actions.
5. Create releases.

## Ghoti-Specific Note

Ghoti is the flagship repo, but this playbook should be reused for NXP, SDA,
Doodle, school/engineering repos, and future polished projects.
