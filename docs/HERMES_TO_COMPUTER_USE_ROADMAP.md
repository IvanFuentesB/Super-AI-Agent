# Hermes To Computer-Use Roadmap

Hermes matters because Ivan wants long boring tasks to become supervised workflows without burning paid credits. The safe path still has stages.

## Now

- Gemma handles guarded offline text tasks.
- Hermes is a manual bridge with safe probes only.
- UI-TARS remains observation-only.
- No live API calls.
- No browser automation.
- No computer-use click/type.

## Next

N+6.3A should build a safe observation harness for the Apple comparison test. It should use manual approval for every click/type and keep all output local.

## Later

Hermes, Browser Harness, Vercel agent-browser, UI-TARS, and Gemma may cooperate only after audited milestones prove:

- no bot/CAPTCHA/cloak bypass
- no fake behavior
- no login/cart/purchase/account action without explicit approval
- no credential/session scraping
- no autonomous posting
- no money/trading/legal action
- no uncontrolled click/type

The first useful target is observation and local summarization, not live account automation.
